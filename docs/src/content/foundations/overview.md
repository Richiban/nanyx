This section is about the **design of Nanyx itself**: language semantics, guiding principles, and compiler/language trade-offs.

It is **not** primarily about day-to-day usage patterns as an application developer.

Use this section when you want to understand questions like:

- Why does the language behave this way?
- What design principles guide feature decisions?
- How does the compiler approach correctness and performance?

If you are looking for practical programming guidance, start with:

- [Getting Started](../getting-started/installation)
- [Language Tour](../tour/variables)
- [Standard Library](../stdlib/standard-library)

Meta docs are especially useful for contributors, language/tooling authors, and anyone evaluating the language model in depth.
